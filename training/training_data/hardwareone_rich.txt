Q: Does the presence sensor work through walls
A: No. The STHS34PF80 uses infrared which requires line of sight. Walls and thick objects block detection.

Q: Can I trigger an action when someone is detected
A: Yes. Create an automation rule like IF PRESENCE>0 THEN ledcolor green. The presence sensor feeds into the automation engine.

Q: Why does the presence sensor give false readings
A: Heat sources like heaters or sunlight can trigger false positives. Check presencestatus and move the sensor away from heat.

Q: What is the difference between presence and motion
A: Presence detects a stationary warm body. Motion detects active movement. The STHS34PF80 has separate channels for each.

Q: How sensitive is the presence sensor
A: Sensitivity depends on the configured thresholds. Check presencestatus to see current levels. Adjust through the web UI settings.

Q: presence sensor keeps triggering
A: Check for heat sources nearby. Run presencestatus to view ambient temperature. Reposition the sensor away from warm objects.

Q: How do I read colour with the APDS
A: Enable color mode with apdsmode color on, then type apdscolor to see red, green, blue, and clear channel values.

Q: Can the gesture sensor detect taps
A: No. The APDS9960 detects directional swipes only: up, down, left, and right. It does not sense tapping.

Q: APDS proximity reads zero
A: Make sure openapds is running and proximity mode is on with apdsmode proximity on. Check i2cscan for the sensor.

Q: How close does something need to be for proximity
A: The APDS9960 proximity sensor works within a few centimetres. Objects further away read as zero proximity.

Q: Can I use gestures to control automations
A: Yes. Read the gesture with apdsgesture and create automation rules that react to the detected swipe direction.

Q: What does openapds do
A: openapds initialises the APDS9960 sensor. After starting, enable the modes you need with apdsmode.

Q: Is the RTC accurate
A: The DS3231 is accurate to about two parts per million, which means it drifts less than a minute per year.

Q: My RTC shows the wrong time
A: Set it manually with rtcset or sync from the internet with rtcsync after connecting WiFi. Check with rtcread.

Q: Can I use the RTC without WiFi
A: Yes. The DS3231 runs on its own battery. Set the time once with rtcset and it keeps ticking independently.

Q: What battery does the RTC use
A: The DS3231 uses a CR2032 coin cell for backup power. It keeps time for years on a single cell.

Q: How often should I sync the RTC
A: The DS3231 is very accurate. Sync once with rtcsync after connecting WiFi and it stays within seconds for months.

Q: Can I use the RTC with automations
A: Yes. Time-based automation rules use the system clock. Sync it from the RTC with rtcsync for accurate scheduling.

Q: What buttons does the gamepad have
A: The Seesaw gamepad has a joystick and several buttons. Use gamepadread to see which ones are currently pressed.

Q: Can the gamepad trigger commands
A: Yes. Gamepad input can navigate the OLED menu and trigger automations bound to button presses.

Q: My gamepad joystick drifts
A: Some joystick drift is normal. Check gamepadread for raw values. If severe, close and reopen with closegamepad then opengamepad.

Q: Does the gamepad work without the OLED
A: Yes. The gamepad provides raw input via gamepadread even without an OLED screen attached.

Q: How do I use the gamepad for menu navigation
A: Start it with opengamepad. The joystick scrolls OLED menu items and the button selects the highlighted option.

Q: gamepad is unresponsive
A: Check i2cscan for the Seesaw chip. Try closegamepad then opengamepad to reinitialise the connection.

Q: GPS takes a long time to get a fix
A: First fix can take minutes, especially indoors. Move to an area with a clear view of the sky for faster satellite lock.

Q: Can I track my location over time
A: Start the GPS with opengps. The web dashboard GPS tab shows your position history on a map with cached tiles.

Q: How do I check satellite count
A: Type gpsread to see satellite count, fix status, latitude, longitude, altitude, and speed.

Q: GPS shows no position
A: Check gpsread for satellite count. Zero satellites means no sky visibility. Move outdoors and wait for a fix.

Q: What data does the GPS provide
A: The GPS reports latitude, longitude, altitude, speed, heading, satellite count, and fix quality. Read with gpsread.

Q: Can I log GPS coordinates
A: Enable GPS with opengps. The device logs positions to the GPS tab in the web UI. Use sensorlog for raw data logging.

Q: How do I enable debug for a specific sensor
A: Type debugmodule 1 where module is the sensor name. For example, debugimu 1 for IMU debug or debugtof 1 for distance.

Q: What debug info goes to the serial console
A: All debug output goes to serial. Use debugmodule 1 to enable and loglevel to control how verbose the output is.

Q: How do I save debug logs for later
A: Type log start to write all debug output to a file. Check log status to see the log path. Stop with log stop.

Q: Can I debug MQTT connections
A: Type debugmqtt 1 to enable MQTT debug logging. Connection attempts, publishes, and errors appear on serial.

Q: debug output is too noisy
A: Use loglevel warn to show only warnings and errors. Or disable individual modules with debugmodule 0.

Q: What I2C address does the sensor use
A: Run i2cscan to see all detected devices and their hex addresses. Each sensor type uses a fixed default address.

Q: Can I change the I2C address
A: Most I2C sensors have fixed addresses. Some have address jumpers. Check the sensor datasheet for options.

Q: I2C scan finds nothing
A: Check wiring: SDA and SCL must be connected. Verify the sensor has power. Run i2cscan again after checking connections.

Q: How many I2C buses does HardwareOne have
A: HardwareOne uses one I2C bus shared by all sensors. Each device needs a unique address on the bus.

Q: What sensors does i2cscan detect
A: i2cscan reports all devices on the I2C bus by address. HardwareOne automatically loads drivers for recognised sensor chips.

Q: Can I see a live thermal image
A: Yes. Start the camera with openthermal. The Sensors tab in the web dashboard shows a live 32 by 24 pixel heatmap.

Q: What resolution is the thermal camera
A: The MLX90640 produces a 32 by 24 pixel thermal image. Each pixel reports temperature in degrees Celsius.

Q: How hot can the thermal camera measure
A: The MLX90640 measures temperatures from minus 40 to 300 degrees Celsius depending on the sensor variant.

Q: thermal image looks noisy
A: Some noise is normal. Multiple readings average out. Make sure the sensor is firmly seated on the I2C bus.

Q: How do I listen to FM radio
A: Type openfmradio to start the receiver, then fmradiotune followed by a frequency like 101.5 to tune in.

Q: What FM frequencies can it receive
A: The RDA5807 receives standard FM broadcast frequencies from 87 to 108 MHz. Tune with fmradiotune.

Q: radio has no sound
A: Check fmradioread for signal strength. Try fmradioseek to find a strong station. Make sure volume is up with fmradiovolume.

Q: How far can the distance sensor measure
A: The VL53L4CX measures distances up to about six metres. Accuracy is best within two metres.

Q: distance sensor reads zero
A: Make sure opentof is running. Check i2cscan for the VL53L4CX. Ensure nothing blocks the sensor lens.

Q: Can I use distance for proximity alerts
A: Yes. Create an automation rule like IF DISTANCE<50 THEN ledcolor red to react when something is within 50mm.

Q: What is the help command
A: help lists all available commands. Type help followed by a command name for detailed usage and examples.

Q: How do I find a specific command
A: Type help to see all commands. Use help followed by a keyword to search for related commands.

Q: I do not know what to type
A: Start with help to see all available commands. Try status for system info or sensors to see active sensors.

Q: What can the LLM do
A: The on-device LLM answers questions about HardwareOne features and commands. It runs entirely on the ESP32-S3.

Q: How do I ask the LLM a question
A: Type llm generate followed by your question. The model processes it locally and prints a short answer.

Q: Is the LLM connected to the internet
A: No. The LLM runs fully on-device in PSRAM. It has no internet access and only knows about HardwareOne.

Q: How do I show something on the OLED
A: Type oledtext followed by your message. Use oledclear to wipe the screen first if needed.

Q: Can I draw on the OLED
A: The OLED supports text lines with oledtext, screen clearing with oledclear, and brightness control with oledbrightness.

Q: OLED screen is blank
A: Type openoled to start the display. Then use oledtext to write a line. Check oledbrightness is above zero.

Q: How much memory is free
A: Type memsample to check current heap and PSRAM usage. memreport gives a detailed breakdown by subsystem.

Q: device is running slow
A: Check memsample for low memory. Unload features you are not using. The LLM model uses the most PSRAM.

Q: Is there enough memory for the LLM
A: Type memsample to check PSRAM. The LLM model needs about 7 MB of free PSRAM to load successfully.

Q: How do I check if MQTT is connected
A: Type mqttstatus to see the MQTT connection state, broker address, and publish interval.

Q: MQTT is not publishing
A: Check mqttstatus for connection state. Verify the broker address with mqttsetbroker. Make sure openmqtt has been run.

ESP-NOW mesh networking extends the range of device-to-device communication by relaying messages through intermediate nodes. Enable relay with espnowrelay. Devices can join named rooms with espnowroomjoin to create logical groups. Messages sent with espnowroomsend reach all room members even across multiple hops.

The map system stores named key-value pairs on the device. Use mapset to save a waypoint or value, mapget to retrieve it, and maplist to see all stored entries. Waypoints can be used with mapnav to calculate routes between named locations.

HardwareOne has a built-in HTTP server that serves the web dashboard. Type httpstart to launch it and httpstop to shut it down. Type httpstatus to check whether the server is running and which port it listens on. The dashboard is accessible from any browser on the same network.

The ban system blocks specific MAC addresses from connecting to the device. Type ban followed by a MAC address to block it. Type banlist to see all banned addresses and banclear to remove all bans. Banned devices cannot connect over WiFi, BLE, or ESP-NOW.

MQTT configuration commands set the broker connection parameters. Use mqttsetbroker for the server address, mqttsetport for the port, mqttsetuser and mqttsetpass for credentials, and mqttsettopic for the publish topic. Use mqttsetinterval to control how often sensor data is published.

BLE settings control the Bluetooth Low Energy radio. Type blesetname to change the advertised device name, blesetpower to adjust transmit strength, blesetinterval to set the connection timing, and blesetmtu to configure the maximum packet size. Use bleautostart on to start BLE at boot.

Edge Impulse runs TFLite machine learning models on the device. Load a model with eifile, start inference with eienable, and run it continuously with eicontinuous. Models can classify audio, motion, or other sensor data. Check results with eistatus and tracking with eitrackstatus.

The image system manages pictures stored on the device. Type imagelist to see all stored images, imageupload to add one from a browser, imagedownload to retrieve one, and imagedelete to remove one. Images are stored on the SD card or internal flash.
