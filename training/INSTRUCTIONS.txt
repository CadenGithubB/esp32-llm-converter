ESP32-S3 On-Device LLM — Training & Converter Package
=======================================================


STEP 1 — Install dependencies (once)
--------------------------------------

    pip install -r requirements.txt

On GPU machines, install PyTorch for your CUDA version first:

    CUDA 12.1:  pip install torch --index-url https://download.pytorch.org/whl/cu121
    CUDA 12.4:  pip install torch --index-url https://download.pytorch.org/whl/cu124

Verify GPU (optional):  python -c "import torch; print(torch.cuda.is_available())"


STEP 2 — Train the model
--------------------------

Use HW1HelpAgent192_deep for best quality: 16 layers, dim=192, 6 heads, FFN=512, seq=128.
Or use HW1HelpAgent for the proven narrow model: 22 layers, dim=128, 4 heads, FFN=768.

GPU training (recommended):

    python train_tiny_model_gpu.py \
        --preset HW1HelpAgent192_deep \
        --text training_data/hardwareone_rich.txt \
        --epochs 250 --lr 3e-4 --batch-size 16 \
        --out ./out_HW1HelpAgent192_deep

CPU training (slow, use only if no GPU available):

    python train_tiny_model.py \
        --preset HW1HelpAgent192_deep \
        --text training_data/hardwareone_rich.txt \
        --epochs 400 --batch-size 8 --lr 3e-4 \
        --out ./out_HW1HelpAgent192_deep

Loss should fall from ~7 down to ~0.1-0.3.
The script runs a Q&A test after training completes.

To check model size before training:

    python train_tiny_model_gpu.py --preset HW1HelpAgent --estimate-only


STEP 3 — Convert to model.bin (browser tool)
---------------------------------------------

3a. Open index.html in Chrome or Edge (Firefox works too).
    No server needed — open the file directly from disk.

3b. Drop the entire output folder (./out_HW1HelpAgent) onto the page, or use
    the file picker to select the files inside it.

3c. Select INT8 quantization, group size 128. These are the defaults.

3d. Click Convert. When finished, click Download.
    This saves model.bin.


STEP 4 — Copy to device
-------------------------

4a. Copy model.bin to the SD card at /sd/llm/ or upload via the web Files page.

4b. Open the LLM tab in the web dashboard, select the model, and click Load.
    Or from the CLI: llm load /sd/llm/model.bin

4c. Expected log output: [LLM] Model ready. PSRAM used: ~6900KB


STEP 5 — Use the LLM
----------------------

Ask questions from the LLM tab in the web dashboard or from the CLI:

    llm generate "What is Hardware One?"

The firmware stops generation after 2 sentences or 80 tokens (configurable).


TRAINING DATA FILES
--------------------

    training_data/
        hardwareone_rich.txt              Complete training corpus (Q&A pairs,
                                          passages, and Do: command pairs).
                                          Edit this file directly to change
                                          training data.

    training_scripts/
        run_all_checks.py                 Run all data quality checks at once
        check_hallucinated_sensors.py     Flag invalid chip names
        check_answer_consistency.py       Verify facts match across answers
        answer_frequency_balance.py       Report answer repetition balance
        find_near_duplicate_answers.py    Find 75%+ word-overlap answer pairs
        answer_fluff_analysis.py          Report verbose phrases and word budget
        topic_coverage_report.py          Count QA pairs per topic
        prose_analysis.py                 Check prose lengths and topic coverage

All Q&A pairs use Q:/A: format. Answers are 30 words or less to fit within
the model's context window.

Note: The GPU trainer masks question tokens during training (only answer tokens
contribute to the loss). This focuses 100% of learning on answer quality.
